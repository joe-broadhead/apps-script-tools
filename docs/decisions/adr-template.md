# ADR Template

- Status: Proposed | Accepted | Superseded
- Date: YYYY-MM-DD
- Owners: @team-or-author
- Related: issue/PR/doc links

## Context

What problem are we solving? What constraints matter?

## Decision

What exactly is being chosen?

## Alternatives Considered

1. Option A
2. Option B
3. Option C

## Consequences

### Positive

- Benefits expected from this decision.

### Negative

- Risks, costs, or new maintenance burden.

### Follow-up

- Tasks needed after acceptance.

## Rollout / Validation

- Tests, checks, and observability signals proving correctness.
